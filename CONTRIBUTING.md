# Contributing to TEL-OS

We welcome contributions from the community to enhance the safety and effectiveness of the TEL-OS governance framework. This document outlines the process for contributing to the project.

## How to Contribute

There are several ways you can contribute to TEL-OS:

### 1. Vector Extraction for New Models
One of the most valuable contributions is extracting rejection vectors for additional models beyond our current support. Our framework currently focuses on Llama-3.1-8B-Instruct, but we aim to expand compatibility.

To contribute new vectors:
- Follow the methodology outlined in our whitepaper
- Submit experimental results demonstrating effectiveness
- Include metadata about the model and extraction process

### 2. Code Improvements
- Bug fixes
- Performance optimizations
- New features that enhance safety without compromising utility
- Better integration with popular ML frameworks

### 3. Documentation
- Improving existing documentation
- Adding examples and tutorials
- Translating documentation to other languages

### 4. Experimental Validation
- Reproducing our experimental results
- Conducting new safety evaluations
- Testing against emerging attack vectors

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or fix
3. Make your changes
4. Add tests if applicable
5. Submit a pull request with a clear description of your contribution

## Code Style

Please follow the existing code style in the repository. For Python code, we follow PEP 8 guidelines.

## Questions?

If you have questions about contributing, please open an issue for discussion.
